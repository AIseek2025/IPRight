# IPRight 代码审计报告（2026-05-08）

## 审计范围
- 后端 API、文档生成链路、任务编排阶段处理器
- 以可复现失败用例与全量测试结果为主线进行问题定位与修复

## 审计方法
- 执行测试：`python3 -m pytest tests -q`
- 针对失败点定向回归与代码审查
- 修复后再次全量回归
- 对修改文件执行诊断检查（无新增诊断错误）

## 发现问题与修复明细

### 1) API 注解兼容性问题（已修复）
- **风险等级**：高
- **问题描述**：`uuid.UUID | None` 在 Python 3.9 + FastAPI/Pydantic 的注解求值中会触发类型解析异常，导致测试收集阶段失败。
- **修复动作**：将相关注解调整为 `Optional[uuid.UUID]`，恢复低版本解释器兼容性。
- **影响范围**：任务工件与截图查询接口参数解析、构建 ID 推导函数。

### 2) 工件/截图按构建过滤兼容性缺陷（已修复）
- **风险等级**：高
- **问题描述**：当任务存在 `active_build_id` 时，接口仅按该构建过滤，导致历史/遗留 `build_id` 为空的记录被错误排除。
- **修复动作**：构建过滤条件改为“当前构建或空构建”并保留上限限制，兼容存量数据。
- **额外修复**：为排序补充稳定次序字段，解决同秒写入导致的返回顺序不稳定问题。

### 3) 说明书中文句式与术语不一致（已修复）
- **风险等级**：中
- **问题描述**：说明书部分默认文案存在多余空格及术语不一致（“访问模式/软件架构”），导致文案规范与测试预期偏差。
- **修复动作**：统一关键句模板，去除中文句中多余空格并规范术语表达。

### 4) 任务画像背景主题偏差（已修复）
- **风险等级**：中
- **问题描述**：背景文本优先使用产品名组合标签，特定场景下无法突出任务关键词主题。
- **修复动作**：背景主题文案优先采用任务关键词，关键词为空时再回退到原策略。

### 5) 截图场景补齐 ID 语义不稳定（已修复）
- **风险等级**：中
- **问题描述**：补齐截图场景使用模块 slug 作为前缀，中文 slug 在跨任务下可读性和可预测性不足。
- **修复动作**：使用模块语义类型前缀（如 `users`）构造补齐场景 ID，增强一致性。

### 6) 源码文档控制字符清洗不足（已修复）
- **风险等级**：中
- **问题描述**：源码文档生成未对 `\x00` 等控制字符形式进行充分规整，可能导致文档输出不可读或不安全字符残留。
- **修复动作**：增加源码行清洗逻辑，同时处理“转义控制字节文本”和“真实控制字符”，统一替换为安全占位字符。

### 7) Plan 阶段对 LLM 强依赖导致失败（已修复）
- **风险等级**：高
- **问题描述**：未配置 LLM Key 时，Plan 阶段直接失败，影响流水线可用性与健壮性。
- **修复动作**：增加模板降级策略：LLM 不可用时自动生成可用 PRD/工单与基础摘要，确保阶段可继续。
